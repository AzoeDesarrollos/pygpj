***Etrucutra básica de los archivos de datos JSON***

Los archivos *.json que se usan en este programa están estructurados de la siguiente manera básica:

{
    "<index>": {
        "Nombre": "<Nombre del dato>",
        "<Dato1>": "<detalles>",
        "<Dato2>": "<detalles>",
        ...
    },
    ...
}

Cada vez que aparezca un numero contenido en un detalle, dicho numero representa el index de uno u otro archivo
json de este programa.

A continuación, las especificaciones sobre el archivo ** armas.json **

    "Tipo": un string de dos caracteres. Puede ser:
        
        "cc": el arma es una de cuerpo a cuerpo
        "ad": el arma es una de ataque a distancia.
    
    "Subtipo": un string de dos caracteres.
        
        "lc": es un arma ligera.
        "um": es un arma que se equipa a una mano.
        "dm": es un arma que se equipa a dos manos
        
        "mu": el arma en sí es munición de un arma de ataque a distancia
        "ar": el arma es una arrojadiza.
        "pr": el arma es una que usa proyectiles para atacar a distancia.
    
    "Competencia": un integral que representa el index de la dote de competencia necesaria para
    utilizar esta arma.
    # debiera decir 'marcial', 'sencilla' o 'exótica' para no genera conflicto con los indices.
    
    "Daño_P": un string contieniendo el daño que esta arma produce cuando es Pequeña. Este detalle
    se imprime. Un arma doble tendrá los dos valores de daño separados por '/'.
    
    "Daño_M": un string contieniendo el daño que esta arma produce cuando es Mediana. Este detalle
    se imprime. Un arma doble tendrá los dos valores de daño separados por '/'.
    
    "Critico": un string conteniendo el ajuste de daño crítico de este arma. Este detalle se imprime.
    
    "Daño_Tipo": un string conteniendo el tipo de daño que el arma produce. Puede ser "Cortante",
    "Contundente" o "Perforante". Cauando el arma pueda causar uno de dos tipos (como una alabarda)
    los tipos de daño estarán separados por 'o'. Si produce dos tipos de daño a la vez (como
    la maza de armas) ambos tipos estarán separados por 'y'.
    
    "Precio": si arma tiene un precio de mercado, estará especificado aquí. Este integral representa el
    valor del arma, en piezas de oro. Un numero decimal aquí representa una fracción de pieza de oro. La
    única 'arma' que no tiene Precio es el Impacto sin armas. Un arma puede tener perfectamente un precio
    de 0, como el bastón o la clava.
    
    "Alcance": solo las armas de tipo "ad" tienen este detalle. Es un string que expresa el alcance, en
    pies (') del arma.
