***Etrucutra básica de los archivos de datos JSON***

Los archivos *.json que se usan en este programa están estructurados de la siguiente manera básica:

{
    "<index>": {
        "Nombre": "<Nombre del dato>",
        "<Dato1>": "<detalles>",
        "<Dato2>": "<detalles>",
        ...
    },
    ...
}

Cada vez que aparezca un numero contenido en un detalle, dicho numero representa el index de uno u otro archivo
json de este programa.

A continuación, las especificaciones sobre el archivo ** armds.json **

    "Tipo": un string que puede ser:
        
        "armd": si el item es una armadura
        "esc": si el item es un escudo.
    
    "Competencia": un integral que representa el index de la dote de competencia necesaria para
    utilizar esta armadura o escudo.
    # debiera decir 'ligera', 'intermedia', 'pesada' o 'escudo' para no genera conflicto con los indices.
        
    "Bon_CA": un integral que expresa el bonificador a la CA que esta armadura o escudo proporciona.
    
    "Bon_max_des": un integral que representa el máximo modificador de Destreza que se aplicará a la CA cuando
    se use esta armadura o escudo.
    
    "Penalizador": las habiliadades que sufren del penalizador de armadura restan este numero de su modificador.
    
    "Velocidad_30": la velocidad modificada de una criatura que normalmente tenga una velocidad táctica terrestre de
    30' cuando use esta armadura.
    
    "Velocidad_20":la velocidad modificada de una criatura que normalmente tenga una velocidad táctica terrestre de
    20' cuando use esta armadura. 
    
    "Precio": si la armadura o escudo tiene un precio de mercado, estará especificado aquí. Este integral
    representa el valor de la armadura o escudo, en piezas de oro. Un numero decimal aquí representa una
    fracción de pieza de oro.
